# V3-Value: A CLV/+EV De-Vig Engine as the Third Pick-Generation Model

A third betting model, **V3-Value** (`sports-betting-value`, tracker id `v3-value`), is added alongside V1-Trends and V2-Sharp. It logs a pick only when an available book price beats the de-vigged Pinnacle (`book_id` 2) fair line by a threshold (≥2% game lines / ≥3% player props) — a new canonical **Line-Value Edge** (`clv_value`). This operationalizes ADR 0003's mandate to optimize for CLV-Positive +EV Candidates, which neither existing model does: V1 is trends/consensus, and V2's RLM thesis is unsourceable on the current data layer (ADR 0006).

**Considered Options:**
- **New third model V3-Value (chosen).** Honors the request for a new model, keeps a genuinely distinct methodology, and preserves the V1-vs-V2 head-to-head experiment.
- **Rebuild V2-Sharp into the CLV engine.** Rejected: edits a graduated skill, orphans V2's existing sample, corrupts the clean V1-vs-V2 comparison, and conflates "build the value engine" with "retire the zombie V2" into one irreversible move.
- **A CLV gate/overlay on V1+V2 output only.** Rejected: the de-vig engine is the pick-*discovery* alpha, not merely a filter; an overlay only ranks a pool that V1 (trends) and V2 (dead RLM) built with the wrong methods, and never scans the board for price-beats-fair.

**Decision:** V3-Value is a Scheduled Run under the ADR 0003 contract (canonical `primary_edge_type` + structured `source_evidence`, market-confirmed). Its engine de-vigs Pinnacle from the BettingPros `/offers` feed — multiplicative for two-way ML/props, power method for spreads/totals — ranks candidates by projected CLV%, and fires above threshold. Originating signals are `clv_value` and `cross_book_gap`; `steam` and consensus `pm_win_pct` are confirmation-only, never standalone. It is sized by fractional Kelly (capped at 2u) and calibrated by CLV (CLV+ rate, average CLV), explicitly **not** by short-run win rate — at the sample sizes in play, win rate is statistically indistinguishable from a coin flip, while CLV confirms an edge in hundreds rather than thousands of bets. It runs on the residential home-PC host (ADR 0006) because the cross-book `/props` and `/offers` calls it depends on are IP-reputation-blocked from the Anthropic datacenter range.

**Cap change:** the Daily Pick Cap rises from 5 to 7 total, adding a `≤3 V3-Value` lane alongside the existing `≤3 V1-Trends` / `≤3 V2-Sharp`. The cap is a flood-guard; a selective CLV model is expected to pass far more often than it bets.

**Consequences:** the tracker `model` field gains `v3-value` and the dashboard becomes a three-way comparison; ADR 0002's edge taxonomy gains `clv_value`. V2-Sharp's zombie state (unsourceable RLM) is left as a separate, deliberate follow-up rather than resolved here. ROI claims for V3 remain provisional until CLV Coverage reaches 90% (ADR 0003); the model is incubated in the `experiments` lab and hardened against an autoeval fixture set before graduation.
